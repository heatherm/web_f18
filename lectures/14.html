Transition

-> Make that, gradual transitions

Most style changes are ~16ms to repaint

'transition' gives us a handle to slow it down.

transition: PROPERTY DURATION FN DELAY;

Full list of properties

is it numeric? has a midpoint?

display values like block and inline-block aren’t animatable

There are performance costs with some animations

Animate the opacity of a duplicated background image instead of animating the blur

Blur Transition

transition: color 200ms ease-in 50ms;

Step Functions

transition-duration: 1s, 2s, 3s;
transition-property: all, border-radius, opacity;

Beware: Negative values for transition-duration will invalidate the entire property value.

Generally, the best effects range between 100 and 200 milliseconds

Keyframe animations

                    

@keyframes animation_identifier {
  keyframe_selector {
    property: value;
    property: value;
  }
  keyframe_selector {
    property: value;
    property: value;
  }
}
                    
                

                    

// The from is the 0% mark. The to is the 100% mark

@keyframes fadeout {
    from {
        opacity: 1;
    }
    to {
        opacity: 0;
    }
}
                    
                

                    

@keyframes color-pop {
    0% {
        color: black;
        background-color: white;
    }
    33% {
        color: gray;
        background-color: yellow;
    }
    100% {
        color: white;
        background-color: orange;
    }
}
                    
                

                    
// Apply multiple
div { animation-name: change_bgcolor, bg-shift; }

keyframes don’t say how long the animation takes

                    

// Add timing and iteration counts

div {
   animation-name: change_bgcolor, round;
   animation-duration: 200ms, 100ms;
   animation-iteration-count: 2;
}
